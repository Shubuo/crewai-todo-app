# CrewAI Todo App

Single-agent CrewAI baseline for generating a complete Todo application.

## Goal

This repository is the Phase 1 baseline for a larger multi-agent project.

The current setup uses one agent with the role `Full-Stack Developer` and asks it to generate a runnable Todo App that:

- supports CRUD operations
- persists data with SQLite
- keeps backend and frontend in a single Python file
- can be launched directly with `python todo_app.py`

This baseline helps validate the LLM connection, measure prompt quality, and establish a reference point before moving to a multi-agent architecture.

## Current Baseline

- Project type: single-agent CrewAI crew
- Agent model: `openai/minimax/minimax-m2.7`
- Generated app target: `todo_app.py`
- Expected stack: `Flask + SQLite + inline HTML/CSS/JS`

## Project Structure

```text
src/crewai_todo_app/
  config/
    agents.yaml
    tasks.yaml
  crew.py
  main.py
todo_app.py
```

`todo_app.py` will be generated by the crew when you run the baseline.

## Setup

1. Install Python `>=3.10,<3.14`.
2. Install [uv](https://docs.astral.sh/uv/) if needed:

```bash
pip install uv
```

3. Install project dependencies:

```bash
uv sync
```

If `uv sync` fails on Intel macOS because of an `onnxruntime` wheel issue coming from CrewAI's dependency tree, use an existing CrewAI environment and install the local package there instead.

4. Create your local env file from the example if it does not already exist:

```bash
cp .env.example .env
```

5. Set your OpenRouter credentials in `.env` using the OpenAI-compatible endpoint:

```env
OPENAI_API_KEY=your-openrouter-api-key
OPENAI_API_BASE=https://openrouter.ai/api/v1
OPENAI_MODEL_NAME=openai/minimax/minimax-m2.7
```

## Run The Single-Agent Baseline

Recommended:

```bash
uv run run_crew
```

Alternative if you prefer the CrewAI CLI:

```bash
uv run crewai run
```

Expected result:

- the crew runs one `Full-Stack Developer` agent
- the agent writes a single-file Todo app to `todo_app.py`

## Run The Generated Todo App

After `todo_app.py` is created:

```bash
uv run python todo_app.py
```

Then open the local Flask URL shown in the terminal.

## What The Agent Is Asked To Produce

The baseline task requires the model to generate:

- a single Python file
- a Flask server
- SQLite persistence
- create, read, update, and delete operations
- a simple browser UI for adding, editing, toggling, and deleting todos
- raw Python output only, without markdown fences

## Next Phase

This repository is intentionally monolithic for Phase 1.

Next, the Todo App workflow can be split into multiple specialized agents, for example:

- product planner
- backend developer
- frontend developer
- tester
- reviewer

That will make it possible to compare single-agent and multi-agent performance against the same target application.
